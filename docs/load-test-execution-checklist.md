# MCJ 부하테스트 실행 체크리스트

## 1. 실행 전 공통 확인

- [ ] 대상 브랜치 변경사항이 dev 환경에 배포되었는지 확인
- [ ] 실행 대상 시나리오와 목적을 확정
- [ ] 테스트 일시와 실행자를 기록
- [ ] dev 환경에서 동시 개발 작업 영향 여부를 확인
- [ ] 테스트 중 외부 사용자 영향 가능성을 확인

## 2. 환경 및 접근 정보 확인

- [ ] `MCJ_BASE_URL` 확인
- [ ] `MCJ_ENV_NAME` 확인
- [ ] 일반 사용자 토큰(`MCJ_ACCESS_TOKEN`) 준비
- [ ] 관리자 토큰(`MCJ_ADMIN_ACCESS_TOKEN`) 준비 여부 확인
- [ ] 시나리오별 대상 ID(`MCJ_GROUP_BUY_ID` 등) 확인
- [ ] 운영 조회 시나리오용 연월(`MCJ_REPORT_YEAR`, `MCJ_REPORT_MONTH`) 확인

## 3. 데이터 사전 조건 확인

- [ ] 공동구매 조회 대상 공구 데이터 존재 여부 확인
- [ ] 마이페이지 조회 대상 사용자 데이터 존재 여부 확인
- [ ] 관리자 조회 대상 정산/환불/발주 데이터 존재 여부 확인
- [ ] 시나리오 대상 데이터가 테스트 중 변경되어도 되는지 확인
- [ ] 상태 전이 또는 외부 연동이 포함된 시나리오인지 확인
- [ ] 쓰기성 요청 포함 시 `ALLOW_STATE_CHANGE` 사용 여부 확인

## 4. 모니터링 및 알림 확인

- [ ] Prometheus에서 대상 환경 메트릭 수집 여부 확인
- [ ] Grafana에서 대상 패널 또는 대시보드 조회 가능 여부 확인
- [ ] dev 알림 채널 또는 Alertmanager 라우팅 분리 여부 확인
- [ ] 테스트 중 반복 알림 발생 가능성 확인
- [ ] 테스트 전후 확인할 패널 목록 정리

## 5. 실행 명령 확인

- [ ] 실행할 시나리오 파일 경로 확인
- [ ] `k6 run` 명령어 확인
- [ ] 쓰기성 시나리오 옵션(`RUN_CREATE_ORDER`, `ALLOW_STATE_CHANGE`) 확인
- [ ] summary export 저장 경로 확인
- [ ] stage / VU / duration 설정값 확인
- [ ] 테스트 종료 조건 확인

## 6. 실행 중 확인

- [ ] HTTP 4xx / 5xx 증가 여부 확인
- [ ] 평균 응답시간 / p95 / p99 확인
- [ ] 애플리케이션 로그 에러 여부 확인
- [ ] CPU / 메모리 / GC / DB 병목 징후 확인
- [ ] 외부 연동 구간 오류 여부 확인

## 7. 실행 후 정리

- [ ] `load-tests/results/` 경로에 summary export 저장
- [ ] `docs/load-test-result-template.md` 기준 결과 문서 작성
- [ ] 병목 의심 지점 기록
- [ ] 재실행 필요 여부 판단
- [ ] 후속 개선 필요사항 및 관련 이슈 정리
